# 个人主页 UI 审查与第二轮优化记录（2026 年 7 月）

## 1. 本轮目标与问题逻辑

本轮不是单纯增加图片或更换文案，而是修正首页的“理解路径”：访客应在首屏快速知道刘杨是谁、解决什么工程问题、为什么可信，以及下一步可以查看什么。

审查后将问题分为四个层级：

1. **P0｜语义与事实准确性**：原首屏动图没有说明 `02 Measure` 与 `03 Validate` 的因果关系，头像置于流程内部还会被误读为流程节点；AVIC 经历、CEng/IMechE 表述和机构标识必须准确。
2. **P1｜首屏信息层级**：标题、个人定位、行动按钮与方法框架应形成稳定的阅读顺序，不能让装饰图抢走主要信息。
3. **P1｜视觉与品牌一致性**：照片清晰度、裁切、Logo 版本、品牌色、容器尺寸和留白需要统一。
4. **P2｜长期质量保障**：中英内容、响应式布局、键盘操作、资源完整性和上线后表现应有可重复的自动检查与人工检查。

## 2. 核心设计决策

### 首屏定位

- 主标题改为“将复杂的高维数据转化为值得信赖的工程决策”，从单一“测量”扩展到更符合实际工作的数据、模型和工程决策链。
- 自我介绍明确覆盖热力学循环与性能分析、工业成像与图像分析、可部署 AI/ML 算法，以及工程软件与系统架构。
- 顶部品牌位使用专业头像，替代抽象的 `YL` 字样；头像仅承担身份识别，不再出现在工程流程中。
- LinkedIn 与电子邮件采用带图标的明确文本链接，并保持可见焦点和足够的点击区域。

### “从数据到工程决策”框架

原来的动图被改为静态、可阅读、可追溯的四步框架：

1. 定义工程问题；
2. 获取并确认数据可信度；
3. 验证模型与解释；
4. 决策、交付与监测。

第 02 步检查标定、可追溯性、代表性与不确定性，以确认数据是否可信；第 03 步再检验物理或数据驱动模型能否在明确边界内解释这些合格证据。高质量数据仍可能被错误模型解释，而看似合理的模型若建立在薄弱证据上也无法支撑工程决策；只有两道门槛都通过，结果才进入交付。任一门槛未通过，都应返回工程问题、证据或模型重新检查。此说明在中英文页面保持语义对应。

### 照片与内容叙事

- 骑行照片采用接近原始比例的响应式资源，不再依赖低分辨率放大或不自然裁切。
- 新增克兰菲尔德博士毕业合照和曼彻斯特大学模型飞机照片，作为教育与工程兴趣的补充证据；不公开朋友姓名等无关个人信息。
- 所有照片生成多尺寸 WebP 与 JPEG 回退版本，保留宽高属性，移除 EXIF/GPS 信息，降低布局跳动并保护隐私。

### 机构 Logo 与事实表达

- Siemens Energy 保留官方白色标识，并在其官方深紫色底色上展示，避免错误改色。
- 2016–2021 经历明确为“克兰菲尔德大学 · AVIC 合作项目”；两个机构使用各自独立标识，避免制造不存在的联合品牌锁定组合。
- AVIC 使用官网提供的本地 PNG；University of Liverpool 使用官网完整横向标识，而不是盾徽或错误版本。
- Logo 作为辅助识别信息，正文仍保留完整机构名称；因此即使图片未加载，经历信息仍然可理解。

## 3. 视觉与交互规则

- **层级**：首屏依次为身份/领域、价值主张、具体能力、主要行动、方法框架；装饰元素不得高于核心内容。
- **间距**：使用统一的节奏变量控制区块、卡片和文字间距；移动端优先保持阅读连续性，不机械压缩桌面布局。
- **字体**：英文使用本地 Inter，中文使用可读性稳定的中文无衬线字体；正文行长、行高和小字尺寸以长时间阅读为准。
- **颜色**：站点主色保持克制，机构品牌色只在 Logo 容器内使用，不扩散为页面主题色。
- **可点击性**：文本链接、按钮和移动导航目标至少约 44px；悬停、键盘焦点、当前导航状态和按下状态必须可区分。
- **空间状态**：菜单展开、键盘焦点、页面滚动定位和 `prefers-reduced-motion` 均应有确定行为；流程内容不依赖动画才能理解。
- **点击逻辑**：主要按钮进入代表工作或公开简历；语言切换前往对应语言与对应页面，不把中文 CV 用户送回英文首页。

## 4. 响应式、无障碍与内容校对策略

- 至少人工检查 320px、700px、1020px 和桌面宽度，重点关注首屏换行、流程卡、双 Logo、照片比例与移动菜单。
- 使用语义化标题、真实有序列表、替代文本、跳转链接和可见焦点；装饰性 Logo 使用空替代文本，机构名称由正文提供。
- 支持键盘顺序、200% 缩放、窄屏重排和减少动态效果；信息不通过颜色或动画单独传递。
- 每次发布同时校对英文与中文：结构、时间、机构、职责、技术术语和行动按钮一一对应，但避免逐字硬译。
- 英文使用英式表达；`CEng through IMechE` / `CEng（经 IMechE 评审）` 准确区分 IMechE 评审与英国工程委员会注册；当前职位时间使用 `Present`，不使用 `Now`。
- 对公开内容实行“可验证、可公开、不过度承诺”原则：不发布薪资、电话、内部系统细节或不可核实的头衔。

## 5. 自动验证清单

`npm run check` 负责持续检查：

- 四个中英文主页/CV 页面都含顶部头像和 AVIC 合作信息；
- 中英文首页均包含顺序一致的四步框架及第 02/03 步区别说明；
- 旧英雄文案、旧头像流程组件、错误日期/术语和误导性资质组合不会回归；
- AVIC 与 Siemens Energy 的非批准 SVG 不存在，AVIC 使用官网 PNG；
- 两种语言均引用两张新增旅程照片和完整 Liverpool 横向 Logo；
- `src`、`srcset`、`poster`、CSS URL、页面锚点及其他本地资源全部可解析；
- 文档语言、描述、canonical、hreflang、重复 ID 和隐私禁用项继续通过检查。

发布前还应运行 HTML 校验、JavaScript 语法检查和 `git diff --check`，并对中英文逐段比对。

## 6. 上线后视觉 QA 验收标准

自动检查不能替代真实浏览器判断。部署完成后需要在公开 URL 上再次确认：

1. 桌面与手机首屏是否在五秒内传达身份、领域、价值主张和主要行动；
2. 四步流程在 320px 窄屏、200% 缩放与减少动态效果下是否仍清楚；
3. 头像、骑行、毕业和模型飞机照片是否清晰，主体裁切是否自然，页面加载时是否无明显跳动；
4. Siemens Energy、Cranfield、AVIC、Liverpool 等 Logo 的色彩、比例、清晰度与暗色背景表现是否正确；
5. LinkedIn、电子邮件、CV、语言切换、移动菜单和页内导航的鼠标、触屏与键盘路径是否全部可用；
6. 中英文页面及浏览器打印输出的名称、日期、术语、标点和语义是否最终一致；
7. GitHub Pages 部署、404、canonical、社交分享图和缓存刷新是否正常。

只有公开站点完成以上复核，本轮优化才算完整交付。

## 7. 公开站点最终验收结果（2026 年 7 月 13 日）

本轮发布曾出现一次可复现的中断状态：修复提交已经合并到 `master`，但公开 URL 在正常部署窗口后仍返回旧版首页。核对 GitHub 中的 `public/index.html` 后，确认源文件完整且不是漏传；随后补充“合并不等于上线，必须通过公开 URL 浏览器验收”的发布守则，并用后续提交安全地重新触发 GitHub Pages。公开站点切换后才继续最终验收。

最终结果如下：

1. **首屏与方法框架通过**：英文首页仅保留一套 `Data-to-decision pathway`；旧 `02 Measure` 和唐突的单句解释均不存在。四阶段、两道独立门槛、进入交付的条件以及失败回退逻辑均完整可见。
2. **中英双语通过**：英文首页可切换到中文同位置；中文四阶段、自我介绍、`CEng（经 IMechE 评审）` 与英文语义一致，不含“经 IMechE 注册”等错误表达。
3. **CV 路径通过**：中文首页进入中文 CV，中文 CV 可切换到英文 CV；页面提供浏览器“打印 / 保存为 PDF”能力，不再引用容易过期的静态 PDF；打印内容包含电子邮件与 LinkedIn。
4. **经历、Logo 与照片通过**：Siemens Energy、Politecnico di Milano、Cranfield、AVIC、XAG、Imperial College London 和 University of Liverpool 的线上资源均成功加载，无破图；Cranfield/AVIC 在 2016—2021 经历中同时出现。毕业合照和曼彻斯特模型飞机照片滚动进入视口后加载清晰的响应式资源。
5. **交互与键盘通过**：跳至主要内容链接可用，按 Enter 后焦点落到 `main-content`；LinkedIn、电子邮件、CV、语言切换和页内导航的目标地址正确；焦点样式、至少约 44px 的主要操作目标、移动菜单状态逻辑与减少动态效果规则均保留。
6. **元数据与分享通过**：英文为 `en-GB`、中文为 `zh-CN`；canonical、描述、结构化数据及 1200×630 的中英文 PNG 社交分享图均指向正确线上资源。
7. **运行健康通过**：公开页面所有已完成加载的图片 `naturalWidth > 0`；站点自身来源的浏览器 console error/warning 为 0。浏览器扩展自身产生的日志已排除，不属于网页问题。
8. **自动检查通过**：15 个 HTML 页面、167 个公开文件和全部内部链接通过站点检查；HTML 校验、JavaScript 语法、Open Graph 图片尺寸/体积和 `git diff --check` 均通过。

本轮没有遗留 P0/P1 阻断项。后续可选优化属于 P2：进一步压缩本地中文字体与个别大型 Logo、为 404 页面补充中文入口，以及清理不再使用的旧 CSS 选择器；这些项目不影响当前内容准确性、核心交互或上线可用性。
